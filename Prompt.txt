Prompt para el Diseño y Desarrollo de una Aplicación Web de Gestión de Inventario
1. Objetivo Principal
Crear una aplicación web moderna, responsiva y eficiente para la gestión de inventario y conteo de stock. La aplicación debe permitir a los usuarios autenticados realizar conteos de productos en ubicaciones específicas, utilizando un escáner de código de barras como método principal de entrada de datos.
2. Funcionalidades Clave
* Módulo de Autenticación:
o Inicio de sesión con email y contraseña.
o Cierre de sesión.
o (Opcional) Registro de nuevos usuarios y recuperación de contraseña.
* Módulo de Catálogo (Gestión de Datos Maestros):
o Productos: CRUD (Crear, Leer, Actualizar, Eliminar) para productos. Cada producto debe tener al menos un Número de Parte (PN), una descripción y un código de barras asociado.
o Ubicaciones: CRUD para las ubicaciones del almacén (ej. Estantería A, Pasillo 3, Nivel 2).
* Módulo de Conteo de Inventario (Funcionalidad Principal):
o Sesión de Conteo: Un usuario debe poder "Iniciar" una nueva sesión de inventario y "Finalizarla".
o Selección de Ubicación: Dentro de una sesión, el usuario debe seleccionar la ubicación física que va a contar.
o Entrada de Datos Rápida:
* La interfaz debe tener un campo de texto principal que reciba la entrada de un escáner de código de barras (que funciona como un teclado).
* Al escanear un código, la aplicación debe buscar el producto en la base de datos y mostrar su descripción.
* El usuario introduce la cantidad contada.
* La aplicación guarda el producto, la cantidad y la ubicación en la sesión de conteo actual.
o Modificación: Permitir la corrección de cantidades ya ingresadas.
o Cierre de Ubicación: El usuario debe poder marcar una ubicación como "Cerrada" o "Finalizada", lo que la bloqueará para futuras ediciones en esa sesión, asegurando la integridad del conteo.
* Módulo de Visualización:
o Un panel simple que muestre el estado del conteo actual: qué ubicaciones ya han sido cerradas y cuáles están pendientes.
o (Opcional) Un reporte básico que muestre las cantidades finales por producto.
3. Pila Tecnológica Sugerida
* Frontend:
o Framework: React (con Vite para un entorno de desarrollo rápido).
o Lenguaje: TypeScript para seguridad de tipos.
o Librería de UI: Mantine UI o Material-UI (MUI) por su amplio set de componentes optimizados para aplicaciones de datos.
* Backend y Base de Datos:
o Servicios: Firebase.
o Autenticación: Firebase Authentication.
o Base de Datos: Cloud Firestore para almacenar en tiempo real los productos, ubicaciones y datos del conteo.
4. Diseño y Experiencia de Usuario (UI/UX)
* Diseño: Limpio, minimalista y funcional. El foco debe estar en la velocidad y la reducción de clics.
* Responsividad: El diseño debe ser "mobile-first", garantizando una experiencia de uso excelente en tablets o móviles (que probablemente se usarán en el almacén para escanear) y también ser completamente funcional en escritorio (para tareas de administración).
* Flujo de Trabajo Optimizado: Para el conteo, la pantalla debe ser muy simple: un campo para el código de barras, un campo para la cantidad y un botón de guardar. El foco debe volver automáticamente al campo del código de barras después de cada guardado para agilizar el proceso.
5. Flujo de Usuario Principal (Ejemplo de Conteo)
1. Un operario inicia sesión en la aplicación desde una tablet.
2. Selecciona la opción "Iniciar Nuevo Inventario".
3. Elige la ubicación "ESTANTERIA-01A".
4. La pantalla de conteo aparece, y el cursor ya está en el campo "Escanear Producto".
5. El operario escanea un producto con un escáner USB/Bluetooth. El código de barras aparece en el campo y la app muestra "Producto: Tornillo X-5".
6. El operario teclea la cantidad "150" y presiona "Guardar".
7. El registro se añade a una lista visible en la pantalla. El cursor vuelve al campo de escaneo.
8. Repite el proceso hasta contar todos los productos de esa ubicación.
9. Finalmente, presiona "Cerrar Ubicación" para bloquearla.
